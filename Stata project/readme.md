# 📊 Проектная работа по статистике  
## по дисциплине «Теория вероятностей и математическая статистика»

Данная проектная работа направлена на выполнение полноценного **статистического исследования**: от постановки исследовательского вопроса до интерпретации результатов и защиты проекта.

Главная идея проекта — не просто выполнить отдельные расчёты, а показать полный цикл анализа данных:

```text
постановка вопроса → сбор или выбор данных → описательная статистика → доверительные интервалы → проверка гипотез → интерпретация результатов
```

Проект выполняется в рамках дисциплины **«Теория вероятностей и математическая статистика»** и предполагает самостоятельную работу с реальными, открытыми, учебными или самостоятельно собранными данными.

---

## 🎯 Цель проекта

Цель проектной работы — развить навыки самостоятельного статистического анализа данных.

В ходе выполнения проекта студент должен научиться:

- формулировать исследовательскую проблему;
- выбирать или собирать данные;
- описывать структуру выборки;
- рассчитывать описательные статистики;
- строить доверительные интервалы;
- формулировать и проверять статистические гипотезы;
- выбирать корректные статистические критерии;
- строить содержательные графики;
- интерпретировать результаты простым и понятным языком;
- защищать полученные выводы устно.

---

## 📌 Общая идея работы

Хороший проект должен быть не набором случайных вычислений, а связной исследовательской историей.

Пример логики:

```text
Мы хотим понять, влияет ли X на Y.
Для этого собрали данные.
Сначала описали выборку.
Затем построили доверительные интервалы.
После этого проверили статистические гипотезы.
На основе результатов сделали вывод.
```

# 🧩 Допустимые варианты проекта

## Вариант 1. Анализ существующих данных

В этом варианте студент использует готовый датасет.

Можно использовать данные из следующих источников:

- учебные датасеты;

- представить содержательные выводы.

- сравнение поведения разных групп покупателей.

# 📊 Обязательная структура статистического анализа

## 1. Описательная статистика

Для количественных переменных необходимо рассчитать:

- среднее арифметическое;
- медиану;
- моду;
- минимум;
- максимум;
- размах;
- дисперсию;
- стандартное отклонение;
- коэффициент вариации;
- квартили;
- межквартильный размах IQR.

Для категориальных переменных нужно рассчитать:

- частоты;
- доли;
- проценты.

Для более сильной работы можно дополнительно рассчитать:

- асимметрию;
- эксцесс;
- сравнение среднего и медианы;
- анализ выбросов;
- форму распределения.

---

## 2. Точечные оценки

В работе должны быть представлены точечные оценки параметров генеральной совокупности.

Примеры:

- оценка математического ожидания через выборочное среднее;
- оценка дисперсии;
- оценка стандартного отклонения;
- оценка доли;
- оценка параметров распределения методом моментов;
- оценка параметров методом максимального правдоподобия.

Пример записи:

```math
\bar{x}
```

является точечной оценкой математического ожидания:

```math
\mu
```

Выборочная доля:

```math
\hat{p}
```

является оценкой генеральной доли:

```math
p
```

---

## 3. Доверительные интервалы

В проекте обязательно должен быть построен хотя бы один доверительный интервал.

Для высокой оценки желательно построить 2–3 интервала, связанных с гипотезами проекта.

| Что оцениваем | Какой интервал использовать | Когда применять |
|---|---|---|
| Среднее одной выборки | z-интервал или t-интервал | Количественная переменная |
| Доля | z-интервал для доли | Бинарная переменная |
| Дисперсия или стандартное отклонение | Интервал через хи-квадрат | Если важно оценить разброс |
| Разность средних | t-интервал или интервал Уэлча | Сравнение двух групп |
| Средняя разность | Парный t-интервал | Данные до/после |
| Разность долей | z-интервал для разности долей | Сравнение частот в двух группах |

Важно не просто записать границы интервала, а объяснить смысл результата.

Пример интерпретации:

> С доверительной вероятностью 95% можно считать, что истинное среднее значение показателя в генеральной совокупности находится в данном интервале.

---

## 4. Проверка статистических гипотез

В проекте необходимо проверить **не менее двух статистических гипотез**.

Каждая гипотеза должна быть оформлена по схеме:

1. Сформулировать `H0` и `H1` словами.
2. Записать гипотезы в математической форме.
3. Указать уровень значимости.
4. Обосновать выбор критерия.
5. Рассчитать тестовую статистику.
6. Найти `p-value` или критическую область.
7. Принять решение по `H0`.
8. Дать содержательную интерпретацию результата.

# 🧭 Рекомендации по выбору критерия

| Исследовательская ситуация | Подходящий метод | Комментарий |
|---|---|---|
| Среднее одной группы сравнивается с нормативом | Одновыборочный t-тест | Если σ неизвестна |
| Средние двух независимых групп | t-тест Стьюдента или Уэлча | Уэлч — если дисперсии не равны |
| До/после у одних и тех же людей | Парный t-тест | Сначала считаются разности |
| Доли или частоты в группах | χ²-критерий или z-тест для долей | Для малых частот — точный критерий Фишера |
| Связь двух количественных шкал | Корреляция Пирсона или Спирмена | Спирмен — при нарушении нормальности |
| Сравнение двух групп без нормальности | Манна–Уитни | Непараметрическая альтернатива t-тесту |
| Сравнение нескольких групп | ANOVA или Краскел–Уоллис | Выбор зависит от нормальности |

# 📈 Визуализация

В проекте должно быть минимум **2–3 графика**.

Графики должны помогать отвечать на исследовательский вопрос, а не просто занимать место.

Можно использовать:

- гистограмму;
- boxplot;
- столбчатую диаграмму;
- круговую диаграмму;
- диаграмму рассеяния;
- график доверительного интервала;
- error bars;
- сравнительный boxplot по группам.

Под каждым графиком должна быть короткая интерпретация:

- что видно на графике;
- есть ли выбросы;
- отличаются ли группы;
- согласуется ли график с результатами проверки гипотез;
- какой вывод можно сделать.

# ⚠️ Что снижает оценку

Оценка снижается, если:

- нет печатного отчёта;
- гипотезы не связаны с данными;
- есть только графики без статистической проверки;
- нет доверительных интервалов;
- не указаны H0, H1, α, p-value или критическая область;
- выводы формальные;
- работа выглядит полностью сгенерированной без понимания;
- отсутствует приложение с анкетой или исходными данными.

# 📌 Итоговый принцип

Качество важнее объёма.

Хороший проект — это не набор случайных расчётов, а связная статистическая история:

```text
вопрос → данные → метод → результат → вывод
```

Проект должен показать, что студент понимает не только формулы, но и смысл статистического исследования.
