# 🔎 Elasticsearch

## Что такое Elasticsearch

**Elasticsearch** — это высокомасштабируемая распределённая поисковая система для полнотекстового поиска и анализа данных.

Elasticsearch работает с неформализованными JSON-документами, предоставляет REST API и часто используется для:

- полнотекстового поиска;
- анализа логов;
- анализа журналов приложений;
- автозавершения;
- умного ввода текста;
- аналитической обработки больших массивов данных.

---

## Elastic Stack

Elasticsearch является ядром экосистемы **Elastic Stack**.

В неё входят:

- **Beats** — одноцелевые отправители данных;
- **Logstash** — механизм сбора и обработки данных;
- **Elasticsearch** — индексирование, хранение, поиск и анализ;
- **Kibana** — визуализация, аналитика и управление.

Общая схема:

```text
Beats → Logstash → Elasticsearch → Kibana
```

---

## Elasticsearch как поисковая система

Elasticsearch поддерживает:

- поиск в режиме реального времени;
- горизонтальное масштабирование;
- многопоточность;
- полнотекстовый поиск;
- поиск по всем выражениям во всех документах;
- автоматическую маршрутизацию запросов по шардам.

Главное преимущество Elasticsearch — высокая скорость и качество обработки текста.

---

## Анализатор текста

Основой полнотекстового поиска является **анализатор**.

Он обрабатывает текст в несколько этапов:

1. **Символьные фильтры**  
   Убирают, добавляют или заменяют отдельные символы.

2. **Токенизатор**  
   Очищает текст от знаков препинания и разбивает его на отдельные слова — токены.

3. **Фильтры токенов**  
   Добавляют, удаляют или изменяют токены.

После обработки текст превращается в набор токенов, который помещается в индекс.

---

## Индексы и шарды

Elasticsearch ищет данные не напрямую по документам, а по индексам.

Индексы могут делиться на сегменты — **шарды**.

Шарды позволяют:

- распределять данные по узлам;
- ускорять поиск;
- масштабировать кластер горизонтально;
- повышать отказоустойчивость.

---

## Elasticsearch как хранилище данных

Elasticsearch также можно рассматривать как нереляционное хранилище документов.

Особенности:

- данные хранятся в формате JSON;
- строгая структура не обязательна;
- доступ осуществляется через REST API;
- данные легко индексируются;
- можно масштабировать кластер на десятки и сотни узлов.

---

## Примеры использования Elasticsearch

Elasticsearch применяется:

- для поиска по сайту;
- для поиска по коду и репозиториям;
- для анализа логов;
- для мониторинга инфраструктуры;
- для поиска по продуктам;
- для поисковых подсказок;
- для визуализации и анализа метрик.

Примеры сценариев:

- поиск вопросов и ответов;
- поиск по репозиториям кода;
- поиск по транзакциям;
- хранение и анализ логов Kubernetes;
- поиск товаров в интернет-магазине;
- мониторинг сервисов и пользовательских инцидентов.

# 🧠 Redis

## Что такое Redis

**Redis** — это высокопроизводительная NoSQL-СУБД с открытым исходным кодом, работающая с данными типа `ключ — значение`.

Redis хранит пары:

```text
key → value
```

где ключ — это идентификатор, а значение — число, строка, список, объект или другая структура данных.

---

## Особенности Redis

В Redis вместо SQL используются скрипты на языке **Lua**.

Redis поддерживает разные типы данных:

- строки;
- списки;
- хеш-таблицы;
- множества;
- упорядоченные множества;
- битовые массивы;
- другие структуры.

---

## Почему Redis быстрый

Redis хранит данные в оперативной памяти.

Благодаря этому он обеспечивает очень высокую производительность — вплоть до миллионов операций в секунду.

---

## Ограничения Redis

У Redis есть два важных ограничения:

1. Объём данных не может превышать объём оперативной памяти.
2. При сбое сервера данные, которые не успели сохраниться на диск, могут быть потеряны.

Поэтому Redis не стоит использовать как единственное основное хранилище критически важных данных.

---

## Где используется Redis

Redis применяется для:

- серверов очередей;
- механизма `publish / subscribe`;
- кеширования результатов запросов к основной БД;
- кеширования промежуточных результатов обработки данных;
- хранения пользовательских сессий;
- счётчиков просмотров;
- счётчиков пользователей онлайн;
- временных данных с высокой скоростью доступа.

# 📨 Apache Kafka

## Что такое Kafka

**Apache Kafka** — это отказоустойчивая распределённая стриминговая платформа с открытым исходным кодом.

Kafka позволяет:

- хранить большие объёмы данных;
- обрабатывать потоки событий;
- доставлять сообщения в реальном времени;
- строить надёжные конвейеры данных;
- связывать множество сервисов между собой.

---

## Kafka как брокер сообщений и хранилище

Kafka сочетает свойства:

- распределённой базы данных;
- брокера сообщений;
- системы потоковой обработки;
- журнала событий.

Kafka собирает данные от приложений, хранит их в распределённом хранилище и отдаёт потребителям по подписке.

---

## Основные понятия Kafka

### Producer

**Producer** — сервис или приложение, которое отправляет сообщения в Kafka.

### Consumer

**Consumer** — сервис или приложение, которое читает сообщения из Kafka.

### Broker

**Broker** — узел Kafka-кластера, который хранит и обслуживает данные.

### Topic

**Topic** — категория или канал сообщений.

Producer отправляет сообщения в topic, а consumer подписывается на topic и читает данные.

### Partition

**Partition** — раздел внутри topic.

Разделы позволяют распределять сообщения и масштабировать обработку.

---

## Журнал сообщений

Kafka хранит сообщения в журнале.

Особенности журнала:

- записи можно только добавлять;
- записи нельзя изменять;
- записи нельзя удалять вручную как обычные строки;
- чтение происходит последовательно;
- порядок сообщений сохраняется.

---

## Где используется Kafka

Kafka используется в следующих сценариях:

- IoT и IIoT;
- потоковая аналитика;
- финансовые системы;
- социальные сети;
- геопозиционные сервисы;
- телеком;
- онлайн-игры;
- сбор логов;
- передача событий между микросервисами;
- построение data pipeline;
- передача данных в Hadoop / HDFS;
- подготовка данных для машинного обучения.

# 📊 Big Data

## Параметры больших данных

Большие данные часто описывают через несколько параметров.

---

## Volume — объём

Большие данные характеризуются большим объёмом.

В лекции указывается ориентир:

```text
от 150 ГБ данных в сутки
```

---

## Velocity — скорость

Данные постоянно обновляются, поэтому их нужно быстро собирать, передавать и обрабатывать.

Пример:

- онлайн-сервисы;
- данные о движении самолётов;
- события пользователей;
- логи приложений.

---

## Variety — разнообразие

Большие данные могут включать:

- таблицы;
- тексты;
- фото;
- видео;
- аудио;
- файлы разных форматов;
- данные из разных источников.

---

## Veracity — достоверность

Для анализа больших данных важно использовать надёжные источники и объективные методы обработки.

---

## Variability — изменчивость

Потоки данных нестабильны.

На них влияют:

- скорость передачи;
- изменение источников;
- поведение пользователей;
- сезонность;
- внешние факторы.

---

## Value — ценность

Сами по себе данные не создают ценность.

Ценность появляется, когда на основе данных можно:

- принимать решения;
- строить прогнозы;
- находить закономерности;
- оптимизировать процессы;
- улучшать продукты.

---

## Типы больших данных

Большие данные могут быть:

- **структурированными**;
- **частично структурированными**;
- **неструктурированными**.

Примеры:

- структурированные данные: медицинские показатели пациентов;
- частично структурированные данные: файлы разных форматов;
- неструктурированные данные: фото, музыка, сообщения пользователей.

# ⚙️ Как работает технология Big Data

Работа с Big Data включает четыре основных этапа.

---

## 1. Сбор данных

Данные собираются из разных источников:

- социальные данные;
- статистические данные;
- медицинские данные;
- машинные данные;
- транзакционные данные.

На этапе сбора выполняется очистка данных — **Data Cleaning**.

Она нужна, чтобы:

- удалить ошибки;
- отфильтровать нерелевантные данные;
- проверить точность;
- привести данные к нужному формату;
- разметить данные для дальнейшей обработки.

---

## 2. Хранение данных

Для хранения больших данных используются разные подходы.

### DWH

**Data Warehouse** — единое хранилище данных для отчётности и принятия решений.

Данные в DWH обычно проходят процесс:

```text
Extract → Transform → Load
```

или

```text
ETL
```

---

### Data Lake

**Data Lake** — озеро данных.

В отличие от DWH, озеро данных может хранить данные без единого формата и строгой структуры.

Часто используется подход:

```text
Extract → Load → Transform
```

или

```text
ELT
```

---

### СУБД

Для больших данных также используются СУБД:

- реляционные;
- нереляционные;
- колоночные;
- распределённые.

Например, ClickHouse позволяет быстро формировать аналитические отчёты по большим объёмам событий.

---

## 3. Обработка данных

Для обработки больших данных используются распределённые вычисления.

Один из ключевых подходов — **MapReduce**.

Общая идея:

1. задача разбивается на множество подзадач;
2. подзадачи распределяются по узлам;
3. узлы обрабатывают данные параллельно;
4. результаты объединяются.

---

## 4. Анализ данных

Для анализа используются:

- SQL;
- BI-системы;
- машинное обучение;
- нейросети;
- аналитические сервисы;
- визуализации и отчёты.

---

# 🐘 Hadoop

## Что такое Hadoop

**Hadoop** — это платформа с открытым исходным кодом для надёжной, масштабируемой и распределённой обработки больших наборов данных.

Hadoop представляет собой набор утилит и библиотек для работы с данными, распределёнными между несколькими кластерами.

---

## Когда данные становятся большими

К Big Data часто относят данные, объём которых увеличивается хотя бы на:

```text
100–150 ГБ в день
```

Для таких объёмов нужны специальные технологии, основанные на:

- распределённом хранении;
- распределённой обработке;
- горизонтальном масштабировании;
- отказоустойчивости.

---

## Принцип работы Hadoop

Hadoop работает по принципу **MapReduce**.

Когда на кластер поступает большая задача:

1. Hadoop делит её на множество небольших подзадач.
2. Каждая подзадача выполняется на отдельном узле.
3. Узлы обрабатывают данные параллельно.
4. Система собирает итоговый результат.

---

## Основные модули Hadoop

Внутри Hadoop выделяют четыре основных модуля.

---

### 1. Hadoop Common

Набор инструментов для создания инфраструктуры и работы с файлами.

---

### 2. HDFS

**HDFS** — Hadoop Distributed File System.

Это распределённая файловая система для хранения данных на разных узлах.

Особенности:

- хранит данные распределённо;
- дублирует данные для надёжности;
- сохраняет работоспособность при отказе отдельных серверов;
- подходит для неструктурированных данных.

---

### 3. YARN

**YARN** — Yet Another Resource Negotiator.

Это система управления кластером Hadoop.

Она позволяет приложениям использовать вычислительные мощности кластера.

---

### 4. Hadoop MapReduce

Компонент, отвечающий за распределённые MapReduce-вычисления.

---

## Дополнительные компоненты Hadoop

### Hive

Платформа для хранения и запроса больших наборов данных из HDFS.

Использует язык запросов **HQL**, похожий на SQL.

---

### Pig

Инструмент преобразования данных.

Используется для подготовки данных разных форматов.

---

### Flume

Инструмент приёма потоковых данных и отправки их в HDFS.

Часто применяется для сбора логов.

---

### ZooKeeper

Координатор распределённых приложений.

Помогает синхронизировать работу компонентов кластера.

---

## Для чего используется Hadoop

Hadoop нужен, чтобы:

- ускорять обработку данных через параллельные вычисления;
- обеспечивать устойчивость данных;
- хранить резервные копии на разных узлах;
- работать с данными любых типов;
- обрабатывать неструктурированные данные.

Практические задачи:

- создание Data Lake;
- обработка данных из социальных сетей;
- анализ поведения клиентов;
- управление рисками;
- анализ логов;
- реагирование на сбои;
- обработка данных IoT.

# ⚡ Apache Spark

## Что такое Apache Spark

**Apache Spark** — фреймворк распределённой обработки данных, который хорошо подходит для потоковой обработки и задач, где важна скорость.

Главное отличие Spark от Hadoop MapReduce:

```text
Spark хранит промежуточные результаты в памяти
```

а Hadoop MapReduce чаще читает и записывает промежуточные данные на диск.

---

## Почему Spark быстрее

Spark избегает постоянной записи промежуточных результатов на диск.

Это даёт прирост производительности, особенно в задачах:

- машинного обучения;
- потоковой обработки;
- итерационных вычислений;
- интерактивной аналитики.

---

## Библиотеки Spark

Поверх ядра Spark работают библиотеки:

- **Spark SQL** — SQL-подобные запросы к распределённым данным;
- **MLlib** — машинное обучение;
- **Structured Streaming** — потоковая обработка данных в реальном времени;
- **GraphX** — задачи с графами.

---

# ⚖️ Spark vs Hadoop

## Когда лучше Spark

Spark подходит, если:

- нужно анализировать потоковые данные в реальном времени;
- скорость решения задачи критически важна;
- используется много параллельных операций;
- применяются итерационные алгоритмы;
- задачи связаны с машинным обучением.

---

## Когда лучше Hadoop

Hadoop подходит, если:

- нужно обрабатывать очень большие наборы данных;
- объём промежуточных результатов превышает доступную память;
- нужно построить инфраструктуру анализа данных при ограниченном бюджете;
- не требуется немедленный результат;
- задачи связаны с пакетной обработкой;
- нужно много читать и записывать данные на диск;
- анализируются исторические или архивные данные.

# 🟡 Yandex Data Proc

## Что такое Yandex Data Proc

**Yandex Data Proc** — управляемый сервис для создания кластеров Hadoop и Spark в Yandex Cloud.

Он автоматизирует:

- создание кластеров;
- настройку виртуальных машин;
- установку программного обеспечения;
- настройку сети;
- обновление компонентов;
- запуск заданий;
- мониторинг;
- масштабирование ресурсов.

---

## Компоненты Data Proc

В Data Proc могут использоваться:

- **Tez** — улучшения MapReduce;
- **Spark** — распределённая потоковая обработка;
- **Hive** — хранение и запросы к большим данным;
- **ZooKeeper** — координация распределённых приложений;
- **HBase** — распределённая NoSQL-БД на базе HDFS;
- **Sqoop** — передача данных между Hadoop и реляционными БД;
- **Oozie** — управление workflow и заданиями MapReduce;
- **Flume** — сбор и перемещение логов;
- **Livy** — REST-интерфейс для Spark;
- **Zeppelin** — браузерный инструмент анализа и визуализации данных.

---

## Интеграция Data Proc

Data Proc интегрируется с:

- Yandex Object Storage;
- Yandex DataSphere;
- сервисами Yandex Cloud;
- Hadoop;
- Spark;
- Hive;
- системами машинного обучения.

---

## Как устроен кластер Data Proc

Основная сущность Data Proc — **кластер**.

Кластер объединяет:

- вычислительные мощности;
- хранилище;
- сетевую инфраструктуру;
- сервисы Hadoop / Spark.

Кластер состоит из подкластеров.

---

## Подкластеры Data Proc

Подкластеры объединяют хосты с одинаковыми функциями.

Основные типы:

- управляющий подкластер;
- подкластер хранения данных;
- подкластер обработки данных.

Примеры ролей:

- `NameNode`;
- `ResourceManager`;
- `DataNode`;
- `NodeManager`.

---

## Хранение данных в Data Proc

Чтобы не переплачивать за хранение большого объёма данных на вычислительных узлах, Data Proc связан с Object Storage.

Основной объём данных можно хранить в объектном хранилище, а на вычислительных узлах держать только горячие данные.

---

## Сеть в Data Proc

У хостов Data Proc нет публичного IP-адреса.

Для подключения используется виртуальная машина, расположенная в той же облачной сети.

Схема подключения:

```text
Локальный компьютер → ВМ по SSH → Data Proc cluster по SSH
```

# 🛠️ Практическая работа: создание Hadoop-кластера

## Шаг 1. Создать бакет

Сначала необходимо создать бакет в Object Storage.

Он используется для хранения:

- зависимостей заданий;
- результатов выполнения;
- журналов.

---

## Шаг 2. Создать сервисный аккаунт

Для доступа к кластеру нужен сервисный аккаунт.

Минимальная роль:

```text
mdb.dataproc.agent
```

Для автоматического масштабирования также нужны роли:

```text
editor
dataproc.agent
```

---

## Шаг 3. Создать кластер Data Proc

В каталоге Yandex Cloud нужно выбрать сервис:

```text
Data Proc
```

Затем нажать:

```text
Создать кластер
```

---

## Шаг 4. Настроить кластер

При создании кластера нужно указать:

- имя кластера;
- версию образа;
- сервисы Hadoop;
- SSH-ключ;
- сервисный аккаунт;
- зону доступности;
- свойства Hadoop;
- бакет Object Storage;
- сеть и подсеть.

Для работы с Hive необходимо выбрать сервис:

```text
HIVE
```

При выборе Hive также потребуются:

```text
MAPREDUCE
YARN
```

---

## Шаг 5. Включить UI Proxy

UI Proxy нужен для доступа к веб-интерфейсам компонентов Data Proc.

Через UI Proxy можно работать с интерфейсами:

- YARN Resource Manager;
- HDFS NameNode;
- Spark History;
- JobHistory;
- Zeppelin.

---

## Шаг 6. Настроить подкластеры

В кластере есть:

- главный подкластер `Master`;
- подкластер `Data`;
- подкластер `Compute`.

Для подкластеров задаются:

- число хостов;
- класс хостов;
- тип хранилища;
- размер хранилища;
- подсеть;
- масштабирование.

---

## Автомасштабирование Compute-подкластера

Для Compute-подкластера можно включить автоматическое масштабирование.

Data Proc может масштабироваться:

- по внутренней метрике YARN;
- по загрузке vCPU.

По умолчанию используется метрика:

```text
yarn.cluster.containersPending
```

Она показывает, сколько единиц ресурсов требуется заданиям в очереди.

# 🐝 Работа с Apache Hive

## Что такое Hive

**Apache Hive** — это платформа для хранения данных и управления ими в экосистеме Hadoop.

Hive позволяет работать с большими датасетами, сохранёнными в распределённом хранилище.

Hive поддерживает форматы:

- CSV;
- TSV;
- Parquet;
- ORC;
- Avro.

Hive использует SQL-подобный язык запросов — **HQL**.

---

## Запуск Hive CLI

На управляющем хосте нужно выполнить:

```bash
hive
```

---

## Проверка работы Hive

```sql
SELECT 1;
```

---

## Создание внешней таблицы

Пример внешней таблицы с данными о перелётах между городами США:

```sql
CREATE EXTERNAL TABLE flights (
  Year bigint,
  Month bigint,
  FlightDate string,
  Flight_Number_Reporting_Airline bigint,
  OriginAirportID bigint,
  DestAirportID bigint
)
STORED AS PARQUET
LOCATION 's3a://yc-mdb-examples/dataproc/example01/set01';
```

---

```sql
SHOW TABLES;
```

---

## Запрос количества перелётов по месяцам

```sql
SELECT Month, COUNT(*)
FROM flights
GROUP BY Month;
```

- Data Proc позволяет создавать Hadoop/Spark-кластеры без ручной настройки;
- Hive позволяет выполнять SQL-подобные запросы к большим данным;
