# ☁️ Лекция 2 — Виртуальная инфраструктура, сети и балансировка нагрузки

Вторая лекция посвящена практической работе с виртуальной инфраструктурой в **Yandex Cloud**.  
В рамках занятия рассматриваются мониторинг виртуальных машин, типы хранилищ данных, работа со снимками и образами ВМ, настройка облачных сетей, групп безопасности и балансировщиков нагрузки.

---

## 🎯 Цель лекции

Цель лекции — научиться управлять базовыми элементами облачной инфраструктуры и понимать, как обеспечить доступность, безопасность и отказоустойчивость облачных сервисов.

После изучения материала студент должен уметь:

- отслеживать состояние виртуальной машины через мониторинг;
- понимать различия между блочным, файловым и объектным хранилищем;
- создавать снимки дисков и образы виртуальных машин;
- настраивать облачные сети и подсети;
- работать с публичными и внутренними IP-адресами;
- использовать группы безопасности;
- создавать балансировщик нагрузки;
- проверять отказоустойчивость веб-сервиса.

## 🧩 Основные темы лекции

### 1. Мониторинг виртуальной инфраструктуры

В лекции рассматривается мониторинг виртуальных машин в **Compute Cloud**.

С помощью встроенного мониторинга можно отслеживать:

- загрузку процессора;
- операции с диском;
- сетевую активность;
- входящий и исходящий трафик;
- динамику показателей за разные периоды времени.

Мониторинг помогает оценивать состояние ВМ, находить перегрузки и контролировать работу облачной инфраструктуры.

---

### 2. Хранилища данных

В лекции сравниваются три типа хранилищ:

| Тип хранилища | Особенности |
|---|---|
| Блочное | Подходит для дисков ВМ, даёт высокую производительность |
| Файловое | Удобно для работы с файлами и совместного доступа |
| Объектное | Хорошо масштабируется и подходит для неструктурированных данных |

---

### 3. Диски в Yandex Cloud

Рассматриваются основные типы дисков:

- **HDD** — стандартный сетевой диск;
- **SSD** — быстрый сетевой диск;
- **NR SSD** — сетевой диск с повышенной производительностью.

Важно учитывать:

- производительность диска;
- IOPS;
- стоимость;
- отказоустойчивость;
- возможность создания снимков.

---

### 4. Резервное копирование и репликация

В лекции объясняется различие между резервным копированием и репликацией.

**Резервное копирование** используется для восстановления данных после повреждения или удаления.

**Репликация** используется для быстрого восстановления сервиса и повышения отказоустойчивости.

---

### 5. Снимки дисков

Снимок диска — это копия файловой системы диска на определённый момент времени.

Снимки позволяют:

- сохранять состояние диска;
- восстанавливать данные;
- создавать новые виртуальные машины на основе снимка;
- переносить конфигурации между ВМ.

Перед созданием снимка рекомендуется остановить операции записи или остановить ВМ.

---

### 6. Образы виртуальных машин

Образ ВМ — это полная копия структуры файловой системы и данных, находящихся на диске.

Образы используются для:

- быстрого создания новых ВМ;
- распространения заранее настроенного программного окружения;
- стандартизации инфраструктуры.

Создание ВМ из образа обычно происходит быстрее, чем из снимка.

---

## 🌐 Облачные сети и VPC

В лекции рассматривается устройство **Virtual Private Cloud (VPC)** в Yandex Cloud.

Основные понятия:

- **облачная сеть** — аналог локальной сети в облаке;
- **подсеть** — диапазон IP-адресов внутри облачной сети;
- **внутренний IP** — адрес для связи ресурсов внутри сети;
- **публичный IP** — адрес для доступа из интернета;
- **статический маршрут** — правило маршрутизации трафика;
- **группа безопасности** — набор сетевых правил.
