# 📊 Что такое Yandex DataLens

**Yandex DataLens** — это сервис Yandex Cloud для визуализации и анализа данных.

С его помощью можно:

- создавать графики и диаграммы;
- строить интерактивные дашборды;
- собирать бизнес-метрики из разных источников;
- проверять аналитические гипотезы;
- работать с данными внутри команды;
- делиться аналитикой с партнёрами и заказчиками.

---

## Основные возможности DataLens

DataLens позволяет:

- быстро проверять гипотезы с помощью данных;
- собирать ключевые бизнес-метрики на одном дашборде;
- создавать интерактивные отчёты;
- визуализировать данные из баз данных и внешних источников;
- использовать селекторы и фильтры;
- строить геоаналитику;
- публиковать дашборды для внешних пользователей.

# 🔄 Принцип работы DataLens

Работа в DataLens строится по цепочке:

```text
Подключение → Датасет → Чарт → Дашборд
```

---

## Подключение

**Подключение** содержит параметры доступа к источнику данных.

Например:

- адрес сервера;
- порт;
- имя базы данных;
- имя пользователя;
- пароль;
- параметры SSL;
- другие настройки подключения.

---

## Датасет

**Датасет** — это набор данных и описание его структуры.

В датасете задаются:

- поля;
- типы данных;
- названия полей;
- агрегации;
- вычисляемые поля;
- связи между таблицами.

---

## Чарт

**Чарт** — это визуализация данных.

К чартам относятся:

- графики;
- диаграммы;
- таблицы;
- карты;
- картограммы;
- другие визуальные представления данных.

---

## Дашборд

**Дашборд** — это страница с визуальной аналитикой.

Он может состоять из:

- чартов;
- заголовков;
- поясняющих текстов;
- селекторов;
- фильтров;
- других виджетов.

---

## Виджет

**Виджет** — отдельный блок на дашборде, в котором размещается чарт, текст, заголовок или другой элемент.

# 📌 Основные понятия DataLens

## Измерения

**Измерения** — это качественные признаки, по которым данные группируются и анализируются.

Примеры измерений:

- дата;
- месяц;
- город;
- регион;
- категория товара;
- тип доставки;
- название продукта.

В интерфейсе DataLens измерения обычно отображаются зелёным цветом.

---

## Показатели

**Показатели** — это числовые значения, которые можно агрегировать.

Примеры показателей:

- выручка;
- количество заказов;
- число пользователей;
- температура;
- давление;
- скорость ветра;
- сумма продаж.

В интерфейсе DataLens показатели обычно отображаются синим цветом.

---

## Логика анализа

Обычно показатели анализируются в разрезе измерений.

Например:

```text
Средняя температура по городам
```

Здесь:

- `Город` — измерение;
- `Температура` — показатель.

---

# 🧮 Вычисляемые поля

DataLens позволяет создавать **вычисляемые поля**.

Вычисляемое поле — это дополнительное поле, значение которого рассчитывается по формуле.

Вычисляемые поля можно создавать:

- на уровне датасета;
- на уровне отдельного чарта.

---

## Где лучше создавать вычисляемое поле

Если поле нужно использовать во многих чартах, его лучше создать на уровне датасета.

Если поле нужно только для одного графика, его можно создать на уровне чарта.

# 🛠️ Практическая часть 1 — создание датасета

Практическая часть начинается с подключения к кластеру **ClickHouse**, где хранится таблица с погодными данными.

Цель практики — сравнить климатические показатели Москвы и Санкт-Петербурга.

---

## Шаг 1. Создание подключения

1. Запустить кластер ClickHouse.
2. Перейти на страницу кластера.
3. На панели слева выбрать **DataLens**.
4. Нажать **Создать подключение**.
5. Указать имя подключения.
6. Ввести пароль к базе данных.
7. Нажать **Проверить подключение**.
8. Если всё работает, нажать **Создать**.

---

## Шаг 2. Создание датасета

После создания подключения DataLens покажет таблицы из базы данных.

Нужно:

1. выбрать таблицу `db1.Weather`;
2. перетащить её на центральную панель;
3. проверить предпросмотр данных;
4. нажать **Сохранить**;
5. задать имя датасета.

---

## Шаг 3. Подготовка полей

Перед построением графиков важно подготовить данные.

Поля рекомендуется переименовать на русский язык:

| Исходное поле | Новое название |
|---|---|
| `LocalDateTime` | Дата и время |
| `LocalDate` | Дата |
| `Month` | Месяц |
| `Day` | День |
| `TempC` | Температура |
| `Pressure` | Давление |
| `RelHumidity` | Влажность |
| `WindSpeed10MinAvg` | Скорость ветра |
| `VisibilityKm` | Видимость |
| `City` | Город |

# 📈 Практическая часть 2 — создание чартов

## Линейная диаграмма температуры

Первый чарт — линейная диаграмма.

Нужно:

1. нажать **Создать чарт**;
2. выбрать тип чарта **Линейная диаграмма**;
3. перенести поле `Дата` в раздел `X`;
4. перенести поле `Температура` в раздел `Y`.

Такой график помогает не только анализировать данные, но и быстро проверять качество датасета.

Например, по графику можно заметить пропущенные или необычные значения.

---

## Разделение температуры по городам

Чтобы сравнить Москву и Санкт-Петербург:

1. перенести поле `Город` в раздел `Цвета`;
2. для поля `Дата` выбрать группировку по месяцам.

После этого можно сравнить среднемесячные значения температуры по двум городам.

---

## Столбчатая диаграмма средней температуры

Для сравнения среднегодовой температуры строится столбчатая диаграмма.

Нужно:

1. выбрать тип чарта **Столбчатая диаграмма**;
2. добавить поле `Город` в раздел `X`;
3. для поля `Дата` выбрать группировку по годам;
4. добавить фильтр по датам;
5. исключить неполные годы, например 2009 и 2019.

---

## Чарты по скорости ветра

Аналогично создаются два чарта по скорости ветра:

1. линейная диаграмма со среднемесячными значениями;
2. столбчатая диаграмма со среднегодовыми значениями.

# 📊 Практическая часть 3 — создание дашборда

После создания чартов нужно собрать их в единый дашборд.

---

## Шаги создания дашборда

1. На панели слева выбрать **Дашборды**.
2. Нажать **Создать дашборд**.
3. Ввести название.
4. Нажать **Создать**.
5. Нажать **Добавить**.
6. Выбрать **Чарт**.
7. Добавить созданные ранее графики.
8. Настроить размеры и расположение виджетов.
9. Добавить заголовки.
10. Добавить селектор по датам.
11. Переименовать виджеты.
12. Сохранить дашборд.

---

## Что должно быть на дашборде

На дашборде можно разместить:

- среднегодовую температуру по городам;
- среднемесячную температуру по городам;
- среднегодовую скорость ветра;
- среднемесячную скорость ветра;
- селектор по датам;
- заголовки для блоков.

# 🗺️ Геоаналитика в DataLens

DataLens позволяет анализировать данные с географическими координатами.

Геоаналитика помогает:

- анализировать прибыльность бизнеса по районам;
- изучать плотность населения;
- оптимизировать логистические маршруты;
- находить точки с нетипичными значениями;
- анализировать пространственные закономерности;
- накладывать несколько геослоёв одновременно.

---

# 🧭 Внутреннее строение карт

Веб-карты обычно состоят из трёх элементов.

---

## 1. Картоснова

**Картоснова** — базовая карта.

На ней уже нанесены реальные объекты:

- дороги;
- здания;
- улицы;
- адреса;
- населённые пункты;
- ландшафтные объекты.

В DataLens базовой картой являются **Яндекс Карты**.

---

## 2. Дополнительные слои

Дополнительные слои — это данные об объектах или явлениях с географическими координатами.

Примеры:

- пробки;
- транспорт;
- события;
- точки продаж;
- зоны доставки.

---

## 3. Пользовательские слои

Пользовательские слои — это собственные наборы геоданных, которые добавляются на карту для анализа и визуализации.

---

# 📍 Основные виды геоданных

## Точка

Точка — базовый вид геоданных.

Она задаётся парой координат:

```text
широта, долгота
```

Координаты записываются в системе WGS 84.

Пример:

```text
55.678366, 37.555126
```

---

## Линия

Линия проходит через несколько точек.

Линии удобно использовать для:

- маршрутов;
- траекторий движения;
- транспортных линий;
- логистических путей.

---

## Полигон

Полигон — это замкнутая фигура, проходящая через несколько точек.

Первая и последняя точки совпадают.

Полигоны используются для анализа территорий:

- районов;
- зон доставки;
- регионов;
- торговых зон;
- административных границ.

---

# 🗺️ Дополнительные элементы карты

## Легенда

Легенда объясняет обозначения, которые используются на пользовательских слоях карты.

При необходимости легенду можно скрыть.

---

## Подпись

Подпись можно добавить к точкам.

Важно:

```text
в DataLens подписями могут быть только числовые значения
```

---

## Тултип

**Тултип** — подсказка, которая появляется при наведении на объект.

В тултип можно выводить:

- текст;
- числа;
- даты;
- значения из нескольких полей.

---

# 🧱 Геослои в DataLens

**Геослой** — это визуализация показателей по точкам или полигонам на карте.

На одной карте может быть до пяти типов визуализации геоданных.

---

## Геослои из геоточек

К слоям на основе точек относятся:

- точечный слой;
- точечный слой с кластеризацией;
- полилинейный слой;
- тепловой слой.

---

## Геослои из геополигонов

К слоям на основе полигонов относится:

- полигональный слой.

---

# 📌 Виды геослоёв

## 1. Точечный слой

Точечный слой используется для отображения местоположения объектов.

С помощью точек можно показать:

- магазины;
- ДТП;
- пункты выдачи;
- объекты инфраструктуры;
- пользователей;
- события.

Размер точки может отражать значение показателя.

Цвет точки может отражать категорию объекта.

---

## 2. Точечный слой с кластеризацией

Кластеризация полезна, когда на карте много точек, расположенных близко друг к другу.

При уменьшении масштаба точки объединяются в кластеры.

Кластер показывает количество объектов внутри него.

Такой подход помогает:

- избежать визуального перегруза;
- увидеть общую картину распределения;
- выявить зоны концентрации событий.

---

## 3. Полилинейный слой

Полилиния состоит из точек, соединённых в линию.

Используется для отображения:

- маршрутов;
- траекторий;
- перемещений объектов;
- транспортных потоков.

Отрезки полилинии можно окрашивать:

- градиентом по показателю;
- дискретными цветами по измерению.

---

## 4. Тепловой слой

Тепловая карта показывает плотность распределения точек.

Она особенно полезна, если:

- точек много;
- они расположены в небольшой географической области;
- обычные точки перекрывают друг друга.

Тепловая карта помогает выявлять зоны повышенной концентрации объектов или событий.

---

## 5. Полигональный слой

Полигональный слой используется для анализа территорий.

Он помогает сравнивать показатели между:

- регионами;
- районами;
- административными зонами;
- зонами доставки;
- торговыми территориями.

Значение показателя отображается цветом или оттенком полигона.

## Создание дашборда с картой

1. На левой панели выбрать **Дашборды**.
2. Создать новый дашборд.
3. Нажать **Добавить**.

# 📝 Практическая часть лекции

В практической части студент выполняет следующие действия:

1. Создаёт подключение к ClickHouse.
2. Создаёт датасет на основе таблицы `Weather`.
3. Переименовывает поля датасета.
4. Разделяет поля на измерения и показатели.
5. Настраивает агрегацию показателей.
6. Создаёт линейную диаграмму температуры.
7. Добавляет разделение по городам.
8. Группирует дату по месяцам.
9. Создаёт столбчатую диаграмму температуры.
10. Создаёт чарты по скорости ветра.
11. Собирает чарты в дашборд.
12. Добавляет заголовки и селектор по датам.
13. Публикует дашборд по ссылке.
14. Изучает виды геоданных.
15. Создаёт карту с полигональным слоем.
16. Настраивает тултипы, цвета и фильтры.
17. Создаёт геоаналитический дашборд.
18. Добавляет селектор по показателю.

# 📌 Что важно запомнить

После лекции важно понимать:

- датасет описывает структуру данных;

- геоданные могут быть точками, линиями и полигонами;
