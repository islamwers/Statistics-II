# Home Assignment 3  
## VM + JupyterLab + LLM в Yandex Cloud

В данной директории находятся материалы для выполнения **Home Assignment 3** — домашнего задания по развёртыванию виртуальной машины в **Yandex Cloud**, запуску **JupyterLab** на удалённом сервере и подключению **LLM** через **Yandex AI Studio**.

Главная идея задания — собрать небольшой рабочий AI-проект, в котором студент показывает, что умеет не только пользоваться готовыми нейросетями, но и встраивать их в собственную облачную инфраструктуру.

Общая логика задания:

```text
Виртуальная машина → Python / JupyterLab → Yandex AI Studio → LLM → практический результат
```

## Возможные проектные направления

В основном PDF-файле предложены разные варианты проектов:

1. **AI-помощник для анализа данных в Jupyter**  
   Анализ данных через `pandas` / `sklearn`, где LLM помогает интерпретировать результаты.

2. **Анализатор текстов через LLM**  
   Автоматическая разметка текстов по теме, тональности, категории, резюме и срочности.

3. **AI-помощник для Excel / CSV**  
   Загрузка таблицы, анализ данных, графики и текстовые выводы через LLM.

4. **AI-код-ревьюер**  
   Проверка Python-кода, объяснение ошибок и рекомендации по улучшению.

5. **Telegram-бот с LLM**  
   Бот принимает сообщение, отправляет его в LLM и возвращает ответ пользователю.

6. **Сайт-анализатор текста**  
   Веб-интерфейс на Streamlit или FastAPI для анализа текста через LLM.

7. **n8n + LLM workflow**  
   Визуальная автоматизация: webhook, Telegram, Google Sheets, LLM и сохранение результата.

8. **mini-RAG по учебным материалам**  
   Система отвечает на вопросы по загруженным документам и учебным материалам.

9. **Генератор отчёта по ML-модели**  
   ML pipeline, метрики, графики и автоматическое формирование отчёта через LLM.

10. **Telegram-автопомощник / style assistant**  
    Проект повышенной сложности с буферизацией сообщений, логированием, стилевым профилем и risk-check.

Можно выбрать один из предложенных проектов или согласовать собственную тему.

Главное требование к собственной теме:

```text
В проекте должны использоваться ВМ, Python / Jupyter / backend, обращение к LLM и практический результат.
```

## Безопасность

При выполнении задания запрещается публиковать в репозитории:

- настоящие API-ключи;
- токены Telegram-ботов;
- приватные SSH-ключи;
- пароли;
- файл `.env` с реальными значениями.

Для демонстрации структуры переменных окружения необходимо использовать файл:

```text
.env.example
```

Пример:

```env
YANDEX_API_KEY=your_api_key_here
YANDEX_FOLDER_ID=your_folder_id_here
YANDEX_MODEL=qwen2.5-72b-instruct
TELEGRAM_BOT_TOKEN=your_telegram_bot_token_here
```

Файл `.env.example` можно загружать в репозиторий.  
Файл `.env` с реальными ключами загружать нельзя.
